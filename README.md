## Инструкция по запуску

### Требования к окружению
* Node.js (рекомендуется LTS версия)
* Android Studio (для Android) / Xcode (для iOS, только macOS)
* JDK 17 (для Android)
* CocoaPods (для iOS)

### Установка и запуск
1.  **Установка зависимостей:**
    ```bash
    npm install
    # или
    yarn install
    ```
2.  **Установка нативных зависимостей (только iOS):**
    ```bash
    cd ios && pod install && cd ..
    ```
3.  **Запуск приложения:**
    * Android: `npx react-native run-android`
    * iOS: `npx react-native run-ios`

## Архитектура проекта

Приложение построено по принципу **Feature-based Layered Architecture**, что позволяет изолировать логику функциональных модулей от общего ядра системы и упрощает масштабирование.

### Структура директорий:
* **src/features/habits** — Основной функциональный модуль. Содержит компоненты экранов, специфичные хуки (useHabits, useHabitActions) и внутренние сервисы.
* **src/core** — Глобальные типы данных, конфигурация хранилища и базовые интерфейсы.
* **src/store** — Глобальное состояние на базе Zustand с персистентностью через MMKV.
* **src/shared** — Переиспользуемые UI-компоненты, темы, константы и вспомогательные утилиты.

## Ключевые сервисы и логика

### 1. Система уведомлений (NotificationService)
Реализована на базе библиотеки `@notifee/react-native`.
* **Тип триггера:** TimestampTrigger с ежедневным повторением (RepeatFrequency.DAILY).
* **Технические особенности:**
    * Автоматическая проверка разрешений на точные будильники (Exact Alarms) для Android 12+.
    * Использование параметра `alarmManager: true` для обеспечения срабатывания в режимах энергосбережения.
    * Интеграция с утилитой `getNextTriggerDate` для корректного расчета времени следующего уведомления.

### 2. Расчет статистики (StatsService)
Включает алгоритмы для анализа прогресса пользователя:
* **Current Streak:** Расчет текущей непрерывной серии выполнений. Серия считается прерванной, если привычка не была отмечена сегодня или вчера.
* **Completion Rate:** Вычисление общего процента эффективности на основе даты создания привычки и истории выполнений.

### 3. Хранение данных (useHabitStore)
Связка Zustand и `react-native-mmkv`:
* **Производительность:** Использование MMKV обеспечивает синхронный доступ к данным через C++ движок, что значительно быстрее стандартного AsyncStorage.
* **Синхронизация:** При обновлении метаданных привычки (например, названия), активные уведомления автоматически пересоздаются с актуальным заголовком.

## Темизация и UI
Приложение использует динамическую систему тем на основе `useTheme` из `@react-navigation/native`.
* **AppTheme:** Расширенный интерфейс темы, включающий кастомные токены (staticWhite, textSecondary, surface) и специфические состояния прозрачности для выполненных задач.
* **Design Tokens:** Централизованные значения отступов (spacing.ts) и радиусов скругления (borderRadius.ts) для поддержания визуальной целостности.
